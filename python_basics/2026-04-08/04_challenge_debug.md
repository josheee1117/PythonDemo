# Challenge：独立调试挑战

## 目标

下面的 `05_debug_challenge.py` 文件里有 **3 个 bug**。
你的任务是：用 VS Code 的调试功能定位并修复它们。

## 规则

- **不给提示**，不告诉你 bug 在哪一行
- 你需要自己设断点、观察变量、分析输出
- 修复后运行文件，看到 `全部通过！` 即为成功

## 验证方式

```powershell
.\.venv\Scripts\python.exe python_basics\2026-04-08\05_debug_challenge.py
```

成功输出：

```
✓ 测试 1 通过
✓ 测试 2 通过
✓ 测试 3 通过
========================================
全部通过！你已经能独立使用 VS Code 调试了。
========================================
```

## 调试建议

- 先直接运行，看报错信息（错误类型 + 行号）
- 在报错行**附近**设断点，用 F5 启动调试
- 在调试控制台里用 `type(变量名)` 和 `print(变量名)` 检查实际值
- 修一个 bug 就运行一次，不要一次猜三个

## 完成后

想想每个 bug 的根因是什么。如果你能用一句话总结每个 bug 的原因，说明你真正理解了。
