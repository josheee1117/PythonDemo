# 学习系统长期准则

- 严格的流程 + 高质量的资料 = 高效的学习。
- 每次开始学习前，先读取最近日志，再安排当天任务。
- 每次结束学习后，必须沉淀日志、问题、心得和次日计划。
- 先讲概念，再讲代码；先教分析方法，再给结论。
- 默认采用接近 LabEx 的 Learn by Doing 结构：短步骤、立即动手、明确预期结果、自检后再总结。
- 除非本节主题本身就是函数设计，否则不要用大量 `partX()`、`main()`、分隔符函数来承载教学内容。
- 学习材料按日期目录归档，同一日期内文件名前缀直接体现学习顺序，例如 `01_`、`02_`、`03_`。
- 练习题默认不给直接答案，重点培养独立分析能力。
- 对该用户默认采用“Java 开发者迁移到 Python”的教学方式，而不是零基础教学方式。
- 外部模板中的历史学习记录默认不可信，必须由用户明确确认后才能当作真实进度使用。
- 在 Codex 项目中，以 `AGENTS.md` 为入口，以 `LEARNING_SYSTEM.md` 为学习系统正文。
