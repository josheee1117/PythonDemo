# 决策：学习材料改为接近 LabEx 的分步实验式

## 背景

用户反馈当前材料“通过很多 `def` 函数来说明”，教学结构过重，不利于快速上手。
现有 `python_basics/day00/day01` 材料本质上是“打印教学内容的脚本”，不是“直接做实验的学习页”。

## 决策

- 后续正式学习材料默认改为接近 LabEx 的 Learn by Doing 结构。
- 教学节奏采用“最小说明 -> 立刻动手 -> 观察结果 -> 再补充解释”，不先给大段理论文字。
- 主入口优先使用 Markdown 实验页，明确目标、步骤、预期输出、自检方式和总结。
- 配套代码优先提供线性 starter/exercise 文件，让用户直接运行、修改、观察结果。
- 除非主题本身就是函数设计，否则不再用大量 `partX()`、`main()`、分隔符函数包装教学内容。
- 学习材料按日期目录归档，例如 `python_basics/2026-03-26/`。
- 同一日期内文件名以 `01_`、`02_`、`03_` 等前缀体现学习顺序。
- 老的脚本式材料直接删除，不再保留为补充参考。

## 直接影响

- 每日任务优先指向实验页和练习文件，而不是教学打印脚本。
- 单个学习单元要缩短到 10-20 分钟一个步骤，减少一次性铺陈过多概念。
- 在用户真正动手前，解释文字只保留完成当前步骤所必需的信息。
- 练习题仍然不给直接答案，但必须提供明确的自检标准。
- 目录和文件名本身要能表达“日期 + 顺序”，避免再靠口头说明。
