- 严格的流程 + 高质量的资料 = 高效的学习
- 先给最小必要概念，再立刻动手练习
- 先教分析方法，再给答案
- 对熟悉的 Java 概念做迁移映射，不做低价值重复训练

执行约束：

- 正式学习从 Day 1 起默认严格按以上顺序推进，不随意跳课。

## 4. 学习工作流

### 开始学习

触发语义：

- `开始学习`
- `今天开始学习`
- `准备学习`

执行步骤：

1. 读取 `daily_logs/` 中最近日志
2. 提取遗留问题、未完成项、下一步计划
3. 结合当前阶段目标生成今日任务
4. 写入 `daily_tasks/YYYY-MM-DD_第X天_任务.md`
5. 向用户汇报今日重点、预计时长、Java/Python 映射重点

### 结束学习

触发语义：

- `学习结束`
- `今天学完了`

执行步骤：

1. 读取今日任务
2. 回顾完成情况、收获、问题
3. 生成 `daily_logs/YYYY-MM-DD_第X天_日志.md`
4. 写入明日计划
5. 更新阶段状态

### 生成周报

触发语义：

- `生成周报`

执行步骤：

1. 汇总本周日志
2. 生成 `weekly_reports/YYYY-WXX.md`
3. 更新下周目标

### 整理知识库

触发语义：

- `整理笔记`
- `更新知识库`

执行步骤：

1. 扫描日志中的知识点与问题
2. 按主题整理
3. 输出到 `knowledge_base/`
4. 优先沉淀速查表、易错点、Java/Python 对照表

## 5. 学习材料标准

正式学习材料默认包含：

1. 本节目标
2. 操作步骤（每步尽量 5-15 分钟）
3. 可直接运行的示例或 starter 文件
4. 预期输出 / 自检标准
5. 原理解释与 Java/Python 对照
6. 常见错误和陷阱
7. 挑战练习
8. 本节总结

生成规则：

- 默认采用接近 LabEx 的 Learn by Doing 结构：短步骤、立即运行、即时反馈、最后总结
- 正式教学必须以“练中学”为主，文字解释只保留完成当前动作所需的最小必要信息
- 正式材料优先使用 Markdown 实验页 + 对应 starter/exercise 文件
- 每个知识点都要尽快落到一个可执行动作，例如运行命令、修改代码、观察输出、修复错误或完成小练习
- 在第一次动手之前，不应出现大段铺垫式说明；若说明变长，必须拆回步骤化实验
- 除非本节主题本身就是函数，否则不要用大量 `partX()` / `main()` / 分隔符函数包装教学内容
- 学习材料按日期目录归档，例如 `python_basics/2026-03-26/`
- 同一日期内文件名必须使用 `01_`、`02_`、`03_` 这类顺序前缀
- 示例代码优先线性、可直接运行；先改少量代码，再引入抽象
- 概念解释必须服务于当前练习，单次只引入少量新概念
- 新函数必须先讲清输入、输出和使用场景，再使用
- 注释必须解释“为什么”
- 每个文件都要有常见错误部分
- 练习题默认不给直接答案，但要给自检方式
- 代码和解释默认使用中文
